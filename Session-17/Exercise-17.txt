1. Πως μπορούμε να κάνουμε ένα Request σε ένα εξωτερικό API χωρίς να γνωρίζει ολόκληρο το url μας; 

Απ. Μπορούμε να χρησιμοποιήσουμε έναν Proxy Server ο οποίος ενεργεί ως ενδιάμεσος μεταξύ του χρήστη και ενός server 
στο Διαδίκτυο. Ο proxy server λαμβάνει τα αιτήματα πρόσβασης σε κάποια ιστοσελίδα από έναν χρήστη, προσκομίζει την σελίδα
από τον server στο Διαδίκτυο και έπειτα την παραδίδει στον υπολογιστή του χρήστη που ζήτησε πρόσβαση. Ένας proxy server παρέχει
ανωνυμότητα στον χρήστη καθώς οι servers που επισκέπτεται βλέπουν ως client τον proxy server και όχι τον υπολογιστή του χρήστη.



2. Αν κάνω ενα GET request και ζητήσω ενα jpeg image, το response τι content-type πρεπει να εχει;

Απ. Το response θα πρέπει να έχει content-type: image/jpeg.



3. Πως μπορώ να κάνω ένα GET request και το περιεχόμενο του να κρατηθεί στην cache για 1 μέρα; 

Απ. Χρησιμοποιώντας την IMemoryCache και την Set extesion method μπορώ να αποθηκεύσω δεδομένα στην cache για συγκεκριμένο χρονικό διάστημα.
Στην περίπτωση μας, για 1 μέρα.



4. Μπορώ να έχω μια εφαρμογή που κάνει μόνο GET requests; 

Απ. Ναι μπορώ. Η GET είναι μια από τις βασικές μεθόδους HTTP και χρησιμοποιείται για να αποκτήσουμε δεδομένα από έναν server,
χωρίς να αλλάξουμε ή να δημιουργήσουμε νέα δεδομένα.



5. Μπορώ να έχω μια εφαρμογή που κάνει μόνο POST requests; 

Απ. Ναι μπορώ. Η POST είναι και αυτή μια από τις βασικές μεδόδους HTTP και χρησιμοποιείται για να στείλουμε δεδομένα σε έναν 
server ώστε να δημιουργήσει καινούργια ή να τα ενημερώσει.



6. Τι status code περιμένω από ένα API αν δεν έχω στείλει ένα υποχρεωτικό πεδίο; 

Απ. Αν δεν έχω στείλει ενα υποχρεωτικό πεδίο περιμένω από ένα API status code 400: Bad Request.



7. Τι status code περιμένω από ένα API αν δεν υπάρχει επικοινωνία με τη βάση; 

Απ. Αν δεν υπάρχει επικοινωνία με την βάση περιμένω από ένα API status code 500: Internal Server Error, διότι η επικοινωνία της
web εφαρμογής με έναν server με βάση δεδομένων είναι εκτός της σφαίρας του HTTP και τυχόν σφάλματα θα πρέπει να συμπεριληφθούν στον
γενικό κώδικα.



8. Μπορώ να κάνω Login με GET request; 

Απ. Όχι, δεν μπορώ καθώς η GET είναι για να αποκτήσουμε δεδομένα από έναν server και όχι για να στείλουμε δεδομένα. Θα έπρεπε να
χρησιμοποιήσουμε POST request.



9. Ποια είναι η διαφορά ανάμεσα στο PUT και στο POST; 

Απ. Η διαφορά μεταξύ POST και PUT requests είναι ότι τα αιτήματα PUT είναι αδύναμα. Δηλαδή, η κλήση του ίδιου αιτήματος PUT πολλές φορές θα
παράγει πάντα το ίδιο αποτέλεσμα. Αντιθέτως, η επανειλημμένη κλήση ενός αιτήματος POST έχει ως παρενέργειες τη δημιουργία του ίδιου πόρου πολλές
φορές. Για παράδειγμα, η μέθοδος PUT καλείται όταν πρέπει να τροποποιήσουμε έναν μεμονωμένο πόρο, ενώ η μέθοδος POST καλείται όταν πρέπει να 
προσθέσουμε έναν θυγατρικό πόρο. Επίσης, οι αποκρίσεις της μεθόδου PUT μπορούν να κρατηθούν στην cache αλλά της POST δεν μπορούν.



10. Ποια μέθοδο θα πρέπει να χρησιμοποιήσω για να «ανεβάσω» ένα αρχείο;

Απ. Για να «ανεβάσω» ένα αρχείο θα πρέπει να χρησιμοποιήσω την μέθοδο POST.
